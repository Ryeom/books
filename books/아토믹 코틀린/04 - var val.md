
## 핵심 요약

> **`val`(불변)을 기본으로, `var`(가변)는 꼭 필요할 때만 사용하라**

코틀린의 가장 중요한 철학 중 하나. 불변성을 통해 코드의 안전성과 신뢰성을 높인다.

## 주요 내용

### 1. var (가변 변수)

```kotlin
var sum = 0
sum = sum + 2  // 재대입 가능
sum += 3       // 복합 대입 연산자
```

- 값을 변경할 수 있는 변수
- `var 식별자 = 초기화`
- 프로그램 실행 중 변경되어야 하는 값에 사용

### 2. val (불변 변수) - 핵심!

```kotlin
val whole = 11
// whole = 15  // ❌ 오류: val을 재대입할 수 없음
```

- **처음 초기화 후 재대입 불가**
- `val 식별자 = 초기화`
- `val` = value (값, 변할 수 없는 것)
- **표현할 수만 있다면 항상 var 대신 val을 사용하라**

### 3. 왜 val을 우선해야 하는가?

**안전성과 신뢰성**:
- 프로그램이 복잡해질수록 변화하지 않는 식별자가 코드를 더 명확하고 안전하게 만듦
- var 사용을 줄이면 프로그램의 신뢰성이 높아짐

**실전에서**:
- val과 시간을 보내다 보면 var가 필요한 경우가 거의 없다는 것을 알게 됨
- 문제 해결이 너무 복잡해지는 경우에만 var 사용

### 4. 타입 지정 (선택적)

```kotlin
// 타입 추론 (권장)
val n = 1
var p = 1.2

// 명시적 타입 지정 (선택)
val n: Int = 1
var p: Double = 1.2
```

**문법**: `var 식별자: 타입 = 초기화`
