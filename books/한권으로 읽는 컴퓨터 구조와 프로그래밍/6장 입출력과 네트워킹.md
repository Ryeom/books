```ad-summary
이 장은 컴퓨터가 고립된 계산기가 아니라, 외부 세계와 소통하고 데이터를 교환하는 거대한 시스템의 일부임을 설명한다. 스위치나 조명 같은 단순한 저수준 장치 제어부터 시작하여, 아날로그 신호를 디지털로 변환하는 과정, 그리고 전 세계를 연결하는 인터넷 네트워크 기술에 이르기까지 컴퓨터가 외부와 상호작용하는 모든 기술적 계층을 다룬다. 핵심은 컴퓨터 내부의 디지털 논리를 어떻게 물리적인 외부 세계의 아날로그 현상 및 다른 컴퓨터와 신뢰성 있게 연결하느냐는 것이다.
```

### 1. 저수준 입출력과 물리적 제어

컴퓨터가 외부 장치를 제어하기 위해서는 전기 신호를 조작해야 한다. I/O 포트의 레지스터를 통해 전압을 조절하여 LED를 켜거나 버튼의 상태를 읽는다. 이때 기계적인 버튼이 눌릴 때 발생하는 미세한 떨림 현상인 바운스를 해결하기 위해 소프트웨어적으로나 하드웨어적으로 신호를 걸러내는 디바운스 과정이 필수적이다. 또한, 제한된 핀으로 많은 장치를 제어하기 위해 멀티플렉싱 기술을 사용한다. 이는 마치 영화 필름이 빠르게 돌아가면 연속된 동작으로 보이는 것처럼, 사람의 눈이 깜빡임을 감지하지 못할 정도로 빠르게 LED를 번갈아 켜서 동시에 켜진 것처럼 보이게 하는 방식이다.

### 2. 아날로그와 디지털의 변환

실제 세계는 연속적인 아날로그 신호(소리, 빛, 온도 등)로 이루어져 있지만, 컴퓨터는 0과 1의 디지털 신호만 처리할 수 있다. 따라서 이 둘 사이를 변환하는 기술이 필요하다.

- 디지털을 아날로그로(DAC): 디지털 숫자를 전압으로 바꿔 스피커나 화면 밝기를 제어한다.
- 아날로그를 디지털로(ADC): 센서에서 들어온 전압을 디지털 숫자로 바꾼다. 이때 비교기를 사용하여 전압의 높낮이를 판단한다.

이 과정에서 가장 중요한 개념은 샘플링이다. 샘플링은 시간의 흐름에 따라 연속적인 신호를 일정한 간격으로 끊어서 측정하는 것이다. 나이퀴스트 법칙에 따라 원래 신호의 최고 주파수보다 최소 2배 빠르게 샘플링해야 신호를 온전히 복원할 수 있다. 만약 이 속도를 맞추지 못하면 에일리어싱이라는 왜곡 현상이 발생한다.

### 3. 통신 프로토콜의 진화

컴퓨터 간에 데이터를 주고받기 위해서는 약속된 규약인 프로토콜이 필요하다.

- 병렬 통신: 여러 가닥의 선을 사용해 한 번에 많은 데이터를 보내지만, 비용이 비싸고 거리가 멀어지면 신호 동기화가 어렵다.
    
- 직렬 통신: 하나의 선으로 비트를 일렬로 보내는 방식이다. 비용이 저렴하고 장거리 통신에 유리하여 USB나 SATA 같은 현대적인 인터페이스의 표준이 되었다. 이는 마치 넓은 고속도로 대신 좁지만 매우 빠르게 달리는 터널을 뚫는 것과 같다.
    

장거리 통신을 위해 마크-스페이스 신호를 주파수로 변조하는 모뎀 기술이 등장했으며, 이는 전화선을 통해 디지털 데이터를 나르는 기반이 되었다.


### 4. 네트워크와 인터넷

네트워크는 근거리(LAN)에서 시작하여 전 세계를 연결하는 광역 네트워크(WAN)인 인터넷으로 발전했다.

- 이더넷: 여러 컴퓨터가 하나의 회선을 공유하며 통신하는 LAN 기술이다. 데이터 충돌을 방지하기 위해 눈치껏 기다렸다가 다시 보내는 방식을 사용한다.
    
- 인터넷: 전 세계의 네트워크를 하나로 묶은 네트워크의 네트워크다. TCP/IP 프로토콜을 사용하여 데이터 패킷을 목적지까지 안전하게 배달한다. 이때 DNS는 전화번호부처럼 사람이 이해하기 쉬운 도메인 주소를 컴퓨터가 이해하는 IP 주소로 변환해 주는 역할을 한다.
    
### 5. 멀티미디어 데이터 처리

소리, 이미지, 비디오 같은 멀티미디어 데이터는 용량이 매우 크기 때문에 효율적인 처리가 필요하다. 그래픽은 점을 찍는 래스터 방식과 선을 그리는 벡터 방식으로 나뉜다. 대용량 데이터를 저장하고 전송하기 위해 압축 기술이 필수적인데, 원본을 완벽히 복원하는 무손실 압축과, 사람의 감각이 둔감한 부분을 삭제하여 용량을 획기적으로 줄이는 손실 압축(JPEG, MP3 등)이 사용된다. 비디오 압축은 이전 화면과 달라진 부분만 저장하는 움직임 보상 기법을 사용하여 효율을 극대화한다.


## 키워드정리 

### 1. 물리적 상호작용과 저수준 I/O 제어

- 디바운스 (Debounce)
    - 등장 배경: 기계식 버튼을 누를 때 접점이 미세하게 튀면서 여러 번의 신호(바운스)가 발생하는 문제를 해결해야 했다.
    - 필요성/중요성: 버튼을 한 번 눌렀는데 컴퓨터가 여러 번 눌린 것으로 인식하는 오류를 방지하기 위해 필수적이다. 하드웨어적 또는 소프트웨어적(타이머, FIR 필터)으로 처리한다.
    - 본문 정의: 버튼 접촉면의 되튕김(바운스)을 없애는 과정이다.
    - 흐름 연결고리: 직전 키워드 없음 $\to$ 직후 - 버튼 입력을 넘어, 제한된 핀으로 다수의 입출력 장치를 제어하는 멀티플렉싱 기술로 이어진다.

- 멀티플렉싱 (Multiplexing)
    - 등장 배경: 7세그먼트 디스플레이나 키패드처럼 많은 핀이 필요한 장치를 프로세서의 제한된 I/O 포트로 제어하기 위해 핀을 공유할 방법이 필요했다.
    - 필요성/중요성: 디스플레이의 애노드나 캐소드를 묶어 연결하고, 아주 빠른 속도로 번갈아 가며 켜는 방식(잔상 효과 활용)으로 하드웨어 비용과 복잡도를 줄인다.
    - 본문 정의: 여러 신호나 장치가 통신 채널이나 처리 장치를 공유하게 하는 기법이다.
    - 흐름 연결고리: 직전 - 디바운스 $\to$ 직후 - 단순한 켜짐/꺼짐 제어를 넘어 정밀한 위치 정보를 읽어내는 그레이 코드로 논의가 전환된다.

- 그레이 코드 (Gray Code)
    - 등장 배경: 회전축의 위치를 일반 2진수로 인코딩할 때, 기계적 오차로 인해 여러 비트가 동시에 변하면서 엉뚱한 값이 읽히는 배열 오류를 막아야 했다.
    - 필요성/중요성: 각도가 변할 때 오직 한 비트만 달라지도록 설계되어, 과도기적 상태에서도 읽기 오류를 최소화한다. 로터리 인코더에 주로 사용된다.
    - 본문 정의: 연속된 값을 표현할 때 인접한 두 값의 비트 패턴이 하나만 다른 인코딩 방식이다.
    - 흐름 연결고리: 직전 - 멀티플렉싱 $\to$ 직후 - 절대 위치가 아닌 상대적 움직임과 방향을 감지하는 쿼드러처 인코딩으로 발전한다.

- 쿼드러처 인코딩 (Quadrature Encoding)
    - 등장 배경: 마우스나 볼륨 조절 놉처럼 절대적인 위치보다는 '움직임 여부'와 '방향'을 알아내는 것이 더 중요한 입력 장치가 필요했다.
    - 필요성/중요성: 2개의 센서만으로 회전 방향(시계/반시계)과 이동량을 정밀하게 감지할 수 있는 효율적인 방식이다.
    - 본문 정의: 2비트 그레이 코드 패턴이 반복되는 방식으로, 네 가지 상태 변화를 통해 움직임과 방향을 감지한다.
    - 흐름 연결고리: 직전 - 그레이 코드 $\to$ 직후 - (다음 섹션) 센서 데이터를 컴퓨터로 전송하기 위한 통신 프로토콜로 넘어간다.

### 2. 데이터 통신과 프로토콜

- 직렬 통신 (Serial Communication)
    - 등장 배경: 병렬 통신이 거리 제약, 비용(많은 전선), 동기화 문제 등을 가지고 있어, 더 적은 선으로 장거리 통신을 할 방법이 필요했다.
    - 필요성/중요성: 데이터 비트를 한 줄로 세워 시간차를 두고 전송함으로써 비용을 줄이고 장거리 전송 효율을 높인다. USB, SATA 등 현대 통신의 기반이다.
    - 본문 정의: 데이터를 한 선을 통해 서로 다른 시간에 나눠서 보내는 방식이다.
    - 흐름 연결고리: 직전 - 쿼드러처 인코딩 $\to$ 직후 - 직렬 통신의 신호를 더 멀리(전화선 등) 보내기 위한 변조(Modulation) 기술로 이어진다.
    - 잠재적 혼동 개념: 병렬 통신과 혼동하지 않아야 한다. 병렬 통신은 여러 선으로 동시에 데이터를 보내지만 비용이 비싸고 장거리에 불리하다.

- 변조 (Modulation) / 복조 (Demodulation)
    - 등장 배경: 0과 1의 디지털 신호(마크-스페이스)는 전화선 같은 아날로그 매체를 통해 멀리 전송하기 어려워, 파동(Wave)에 정보를 실어야 했다.
    - 필요성/중요성: 반송파(Carrier)의 진폭, 주파수, 위상 등을 조절하여 디지털 정보를 아날로그 신호로 변환해 전송한다. 모뎀(Modem)이 이 역할을 수행한다.
    - 본문 정의: 전달하려는 신호를 바탕으로 반송파를 변화시키는 것이 변조이며, 이를 다시 원래 신호로 되돌리는 것이 복조이다.
    - 흐름 연결고리: 직전 - 직렬 통신 $\to$ 직후 - 변조된 신호를 사용하여 전 세계를 연결하는 네트워크와 패킷 교환으로 확장된다.

- 패킷 교환 (Packet Switching)
    - 등장 배경: 회선 교환(전화) 방식은 통화 내내 회선을 점유하여 데이터 전송에 비효율적이었으므로, 회선을 공유하며 효율적으로 데이터를 나를 방법이 필요했다.
    - 필요성/중요성: 데이터를 패킷이라는 작은 단위로 쪼개고 주소 정보를 붙여 전송함으로써, 하나의 회선을 여러 사용자가 공유할 수 있게 한다. 인터넷의 기반 기술이다.
    - 본문 정의: 통신 내용을 수신자와 송신자 주소가 포함된 패킷으로 나누어 전송하는 방식이다.
    - 흐름 연결고리: 직전 - 변조/복조 $\to$ 직후 - 패킷이 올바른 목적지까지 가도록 돕는 TCP/IP 프로토콜로 이어진다.
    - 잠재적 혼동 개념: 회선 교환(Circuit Switching)과 혼동하지 않아야 한다. 회선 교환은 통신하는 동안 물리적인 회로를 독점하는 방식이다.

- TCP/IP
    - 등장 배경: 서로 다른 네트워크들을 연결하고(인터넷), 패킷이 손실되거나 순서가 뒤바뀌는 문제를 해결하여 신뢰성 있는 통신을 보장해야 했다.
    - 필요성/중요성: 인터넷의 표준 프로토콜이다. IP는 패킷의 주소를 지정해 옮겨주고, TCP는 패킷의 배달과 순서를 보장한다.
    - 본문 정의: 인터넷이 사용하는 두 가지 핵심 프로토콜로, 전송 제어 프로토콜(TCP)과 인터넷 프로토콜(IP)을 의미한다.
    - 흐름 연결고리: 직전 - 패킷 교환 $\to$ 직후 - (다음 섹션) 디지털 컴퓨터가 아날로그 세상을 이해하기 위한 샘플링 기술로 논의가 전환된다.

    ### 3. 아날로그와 디지털의 변환 및 멀티미디어

- 샘플링 (Sampling) / 나이퀴스트 정리 (Nyquist Theorem)
    - 등장 배경: 컴퓨터는 0과 1밖에 모르지만, 실제 세상의 소리나 빛은 연속적인 아날로그 값이므로 이를 디지털로 변환해야 했다.
    - 필요성/중요성: 연속적인 신호를 일정한 시간 간격으로 측정하여 수치화한다. 나이퀴스트 정리에 따라 최고 주파수의 2배 이상으로 샘플링해야 원래 신호를 온전히 복원할 수 있다.
    - 본문 정의: 시간이나 공간상 일정한 간격으로 값을 읽어 아날로그 데이터를 디지털로 바꾸는 과정이다.
    - 흐름 연결고리: 직전 - TCP/IP $\to$ 직후 - 샘플링된 데이터를 다시 아날로그로(DAC) 바꾸거나 아날로그를 디지털로(ADC) 바꾸는 변환기 기술로 이어진다.
    - 잠재적 혼동 개념: 에일리어싱(Aliasing)은 샘플링 속도가 충분하지 않을 때 생기는 왜곡 현상(가짜 신호)이다.

- DAC (Digital-to-Analog Converter) / ADC (Analog-to-Digital Converter)
    - 등장 배경: 디지털 음원을 스피커로 듣거나(DAC), 마이크의 소리를 녹음하는(ADC) 등 컴퓨터와 아날로그 장치 간의 상호작용이 필요했다.
    - 필요성/중요성: DAC는 디지털 숫자를 전압으로 변환하며, ADC는 비교기를 사용하여 아날로그 전압을 디지털 숫자로 변환한다. 플래시 변환기, 램프 변환기, 연속 추정 변환기 등 다양한 ADC 방식이 있다.
    - 본문 정의: 디지털 값을 아날로그 신호로 변환하는 장치(DAC)와 아날로그 신호를 디지털 값으로 변환하는 장치(ADC)이다.
    - 흐름 연결고리: 직전 - 샘플링 이론 $\to$ 직후 - 샘플링된 복잡한 신호(소리 등)를 분석하는 수학적 도구인 푸리에 변환으로 이어진다.

- 푸리에 변환 (Fourier Transform)
    - 등장 배경: 복잡하게 섞인 파형(화음 등)에서 각각의 구성 주파수를 분리해내거나 분석할 필요가 있었다.
    - 필요성/중요성: 모든 파형은 사인파의 합으로 표현될 수 있다는 원리를 이용해, 시간 영역의 신호를 주파수 영역으로 변환한다. MP3 압축이나 이퀄라이저 등에 필수적이다.
    - 본문 정의: 시간에 따른 진폭을 주파수에 따른 진폭으로 변환하여 파형을 분석하는 수학적 방법이다.
    - 흐름 연결고리: 직전 - ADC/DAC $\to$ 직후 - 소리뿐만 아니라 시각 정보(이미지, 비디오)를 처리하는 래스터/벡터 그래픽으로 논의가 확장된다.

- 래스터 그래픽 (Raster Graphics) / 벡터 그래픽 (Vector Graphics)
    - 등장 배경: 컴퓨터 화면에 그림이나 글자를 표현하는 방식에 대한 기술적 접근이 필요했다.
    - 필요성/중요성: 벡터는 선을 그리는 명령어로 이미지를 표현하여 확대해도 깨지지 않으나 실사 표현에 한계가 있고, 래스터는 점(픽셀)의 배열로 이미지를 표현하여 사진 등에 적합하나 데이터 양이 많다.
    - 본문 정의: 벡터 그래픽은 선이나 도형 그리기 명령어로 이미지를 구성하는 방식이고, 래스터 그래픽은 픽셀이라는 점들의 2차원 배열로 이미지를 표현하는 방식이다.
    - 흐름 연결고리: 직전 - 푸리에 변환 $\to$ 직후 - (마무리)
